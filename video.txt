"Hi, this is my submission for the AI-First CRM HCP module - the Log
Interaction screen. The goal is a tool for pharmaceutical field reps to log
their interactions with Healthcare Professionals. What makes it AI-first is
that a rep can log an interaction in two ways: by filling a structured form,
or simply by describing it in plain English to an AI Assistant, which extracts
the details and fills the form for review before saving.

The stack is React with Redux on the frontend, Python FastAPI on the backend,
a LangGraph agent that calls Groq LLMs as the AI layer, and PostgreSQL for
storage, with the Google Inter font throughout."

NOTE TO MENTION:
"The brief specifies gemma2-9b-it, but Groq has decommissioned that model, so I
used llama-3.1-8b-instant. It's configurable via one environment variable."

========================================================================
PART 4 - CODE STRUCTURE EXPLANATION  (9:00 - 13:00)
========================================================================
Open the code editor. Keep it simple: the project has two folders, backend
and frontend. Say: "The repo is split into a Python backend and a React
frontend."

SAY THIS (one line per file, point as you go):

BACKEND (the brain - Python + FastAPI):
  "main.py is the FastAPI server. It has the API endpoints - one to save an
   interaction, one to list them, one to edit, and the /api/chat endpoint for
   the AI assistant."
  "The agent folder is the AI part, built with LangGraph:
     - graph.py builds the LangGraph agent - it decides which tool to run.
     - tools.py has the 5 tools: log, edit, search, sentiment, and suggest.
     - llm.py connects to Groq (the LLM)."
  "models.py defines the database tables, and schemas.py validates the data."

FRONTEND (what the user sees - React + Redux):
  "The components folder has the UI: the form, the AI chat panel, and the
   recent-interactions list."
  "The store folder is Redux - it holds the app's state, like the chat
   messages and the interactions list."
  "api.js is what talks to the backend."

THEN EXPLAIN THE FLOW IN ONE BREATH (this is the important part):
"So the flow is simple: I type a message in the chat. It goes to the backend.
The LangGraph agent reads it and picks the right tool. If I'm describing an
interaction, it pulls out the details with the LLM and sends them back, and
Redux fills the form on screen. I review it, click Log Interaction, and it's
saved to the PostgreSQL database and shows up in Recent Interactions."

ONE-LINE SUMMARY TO END PART 4:
"In short: React shows the screen, FastAPI handles the requests, LangGraph is
the AI brain that runs the tools, and PostgreSQL stores everything."

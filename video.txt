========================================================================
PART 4 - CODE STRUCTURE EXPLANATION  (9:00 - 13:00)
========================================================================
Switch to the code editor. Walk through the repo layout:

BACKEND (backend/):
  - agent/graph.py  -> the LangGraph agent. Show the StateGraph: an "agent" node
    (the LLM bound to the tools) and a "tools" node, with a conditional edge that
    loops back so the LLM can read tool output and reply. This is the core of the
    "agent decides which tool to call" requirement.
  - agent/tools.py  -> the tools. Show log_interaction (LLM entity extraction ->
    prefills the form) and edit_interaction (updates a saved record). Mention
    search_hcp, sentiment_analysis, suggest_next_action, save_interaction.
  - agent/llm.py    -> the Groq client (model from .env).
  - models.py       -> SQLAlchemy tables (HCP, Interaction).
  - schemas.py      -> Pydantic request/response models.
  - main.py         -> FastAPI endpoints:
      POST /api/interactions, GET /api/interactions, PUT /api/interactions/{id},
      POST /api/chat, GET /api/hcps, GET /api/health.
  - seed.py         -> demo HCP data.

FRONTEND (frontend/src/):
  - store/          -> Redux Toolkit: interactionsSlice (CRUD thunks) and
    chatSlice (conversation + the form-prefill draft + save signal).
  - components/     -> Sidebar, LogInteractionForm, ChatInterface, InteractionsList.
  - api/api.js      -> axios client with friendly error handling.
  - App.jsx         -> tab layout; styles/index.css uses the Inter font.

Explain the end-to-end flow:
"When the rep chats, the message goes to POST /api/chat. The LangGraph agent
decides a tool. For logging, log_interaction extracts the fields and returns
them; the backend sends them back as form_prefill; Redux stores them and the
form populates. The rep reviews and clicks Log Interaction (or says 'save it'),
which calls POST /api/interactions and the row is stored in PostgreSQL."
